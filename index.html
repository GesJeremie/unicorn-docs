Introduction

Unicornfm isn’t released yet and is currently in development. If you see that, it means you are a fucking-very early user and you should be proud (and almost nothing listed here will work :p.)

Welcome to the UnicornFM API documentation!

This API is the core of UnicornFM, think of unicornfm.com like a fancy UI wrapper around it. It means you could be able to create your own unicorn app or whatever you could think of, we we are very excited to see what you gonna build with it!

Note - We would love to see a Slack / Messenger bot to push songs …

Authentification

Our API is public, anyone is allowed to access our API, you don’t even have to register for an API key, just start to code right away.

CORS is allowed on the API and the Socket :)

Remember - Don’t be a dick.

API Format

This API is based on the specification of http://jsonapi.org/format. This is a well-known standard and have multiple implementations in different languages/frameworks: http://jsonapi.org/implementations.

The base API endpoint is http://api.unicornfm.com/v1.

Users

If you played with unicornfm.com you should know you are able to create, join a server, push songs without registration. This is possible thanks to our concept of anonymous users. In fact when you browse our app you are automatically “connected” as an anonymous user. Thus we are able to know who is the owner of a server, who pushed a song, etc.

Most of the endpoints of the API will ask for an user token.

The User Object

{
  type: "user",
  id: "f066d4ef-c332-4d2b-a793-65aa88768e55",
  attributes: {
    token: "5976423a-ee35-11e3-8569-14109ff1a304",
    inserted-at: "2017-05-27 20:21:38.987925",
    updated-at: "2017-05-27 20:21:38.987925"
  }
}

Attributes

Attribute Description
token String Auto generated token which will be often asked as a required parameter by other endpoints
inserted-date Timestamp Time at which the user was created
updated-date Timestamp Time at which the user was updated
The token is returned only at the creation of the user.

Create a User

curl --request POST http://api.unicornfm.com/v1/users

this.get('store').createRecord('user').save()

The above command returns JSON structured like this:

[
  jsonapi: {
    version: "1.0"
  },
  data: [
    {
      type: "user",
      id: "f066d4ef-c332-4d2b-a793-65aa88768e55",
      attributes: {
        token: "5976423a-ee35-11e3-8569-14109ff1a304",
        inserted_at: "2017-05-27 20:21:38.987925",
        updated_at: "2017-05-27 20:21:38.987925"
      }
    }
  ]  
]

HTTP Request

POST http://api.unicornfm.com/v1/users

Servers

Servers are the entity we like to call “unicorns”. Users can create/join them and push tracks to them, it’s definitely the core concept of Unicorn.

The Server Object

{
  type: "server",
  id: "69b2d5af-ff4d-4c85-843f-cdf1dcec4241",
  attributes: {
    name: "idiot-parrot-686",
    user-id: "86ef8527-144b-4756-b5da-aa1faadff5ad",
    current-track: "03afecc8-4fb4-4c91-a041-e053f77def86"
    inserted-at: "2017-05-27 20:21:38.987925",
    updated-at: "2017-05-27 20:21:38.987925"
  }
}

Attributes

Attributes Description
name String The name of the server
user-id String The owner of the server
current-track String The track currently played on the unicorn
inserted-date Timestamp Timestamp when the server was created
updated-date Timestamp Timestamp when the server was updated

Create a Server

curl http://api.unicornfm.com/v1/servers \
  -d '{"server":{"user_token":"5976423a-ee35-11e3-8569-14109ff1a304"}}' 

this.get('store').create('server', {
  userToken: '5976423a-ee35-11e3-8569-14109ff1a304'
});

The above command returns JSON structured like this:

[
  jsonapi: {
    version: "1.0"
  },
  data: [
    {
      type: "server",
      id: "740d712c-de24-4131-8ae4-b674a320e228",
      attributes: {
        name: "idiot-parrot-678",
        user-id: "af3c79c7-8300-4c9f-8e02-ed0257a2ab12",
        current-track: null,
        inserted-at: "2017-05-27 20:21:38.987925",
        updated-at: "2017-05-27 20:41:38.987925"
      }
    }
  ]  
]

HTTP Request

POST http://api.unicornfm.com/v1/servers

Data Parameters

Parameter Description
user-token Required The token of the user
name Optional The name to give to the server. If not provided, we auto generate one. Be aware we suglify the input which allow only alpha-numeric characters and dashes. Keep in mind we only accept unique names.

Update a Server

this.get('store').findRecord('server', '13cdd8b1-f3d4-449c-ab61-ff78fed187cb').then(function(server) {
  // Required
  server.set('user-token', 'ad0700ce-b579-40eb-8201-c502611dc9e0');

  // Optional
  server.set('name', 'beautiful-unicorn');
  server.set('user-id', 'bfe48bff-f6ed-4091-a876-a36b7d9a146a');
  server.set('current-track', 'da306b85-2247-44c3-aba0-bd4871482672');
  server.save(); 
});

HTTP Request

PATCH http://api.unicornfm.com/v1/servers/:id

Data Parameters

Parameter Description
user-token Required The token of the owner (current user id). If the token provided doesn’t match the token of the server’s user-id, the API will reject your call.
name Optional The name of the server. Be aware we suglify the input which allow only alpha-numeric characters and dashes. If the name provided is not unique, the API will reject the call.
user-id Optional The owner of the server. If the user-id doesn’t exist, the API will reject the call.
current-track Optional Current track playing on the server. If the track doesn’t exist, the API will reject the call.

Retrieve a Server

curl "http://api.unicornfm.com/v1/server/idiot-parrot-678"

this.get('store').findRecord('server', 'idiot-parrot-678');

The above command returns JSON structured like this:

[

  data: [

      type: "server",
      id: "idiot-parrot-678",
      attributes: {
        name: "idiot-parrot-678",
        user-id: 20,
        current-track: 130,
        inserted-at: "2017-05-27 20:21:38.987925",
        updated-at: "2017-05-27 20:21:38.987925"
      }
    }
  ]  
]

Songs

We don’t store any physical songs on our servers, in fact we retrieve our songs from Youtube. Think of songs like a wrapper of the youtube API with digest results.

We are thinking to use multiple providers in the future such as VIMEO and Soundcloud.

The Song Object

Attributes

Attributes Description
title String Title of the song
thumbnail String Thumbnail url
duration Integer The total duration of the song in seconds
code String The reference id of the song of the provider
provider String The service provider, currently only youtube
inserted-date Timestamp Timestamp when the server was created
updated-date Timestamp Timestamp when the server was updated

Search songs

curl http://api.unicornfm.com/songs?query=haezer+minted

this.get('store').query('songs', {
  query: 'Haezer - Minted'
});

The above command returns JSON structured like this:

[
  jsonapi: {
    version: "1.0"
  },
  data: [

HTTP Request

GET http://api.unicornfm.com/v1/songs

Data Parameters

Parameter Description
query Keyword(s) to find related songs
youtube_api_key optional If we don’t have in cache the results for the search given we hit the youtube API with our API key. If you are not a dick you will provide your api key.

Errors

This error section is stored in a separate file in `includes/_errors.md`. Slate allows you to optionally separate out your docs into many files…just save them to the `includes` folder and add them to the top of your `index.md`‘s frontmatter. Files are included in the order listed.

The Kittn API uses the following error codes:

Error Code Meaning
400 Bad Request – Your request sucks
401 Unauthorized – Your API key is wrong
403 Forbidden – The kitten requested is hidden for administrators only
404 Not Found – The specified kitten could not be found
405 Method Not Allowed – You tried to access a kitten with an invalid method
406 Not Acceptable – You requested a format that isn’t json
410 Gone – The kitten requested has been removed from our servers
418 I’m a teapot
429 Too Many Requests – You’re requesting too many kittens! Slow down!
500 Internal Server Error – We had a problem with our server. Try again later.
503 Service Unavailable – We’re temporarily offline for maintenance. Please try again later.
